All About Animations

What is a CSS Animation?

CSS animations make it possible to animate a tansition from one CSS style to another. The animation consists of two parts - a style describing the CSS animation and a set of keyframes that describe the start and end states of the animations style.

Why Use a CSS Animation?

  • Easy to create basic animations (no JS required!)
  • Better performance - the browser can optimize for performance and efficiency

How Do We Create CSS Animations

To create a CSS animation, you first need to configure the timing, duration and other dwtails of how the animation sequence should progress. This is done using the animation property and its subproperties.Next, use the @keyframes at-rule to configure the actual appearance of the animation. Each keyframe in @keyframe at-rule describes how the animated element should render at a given time during the animation. You can even add multiple animations to one element by separating values with a comma. Keep in mind, however, that a little goes a long way! Adding too many animations can be distracting and overwhelming for the user and can actually result in a poor user experience. Too many animations can also negatively impact web performance.

Transitions, Animations & Accessibility

It's also important to consider accessibility when adding animations and transitions to your webpages.Blinking and flashing anumation can be problematic for people with ADHD and certain kinds of motion can be a trigger for Vestibular disorders, epilepsy and migraine and Scotopic sensitivity. Providing the ability to reduce animations or switch off animation based on the user's preference is important. We can use the prefers-reduced-motion media query to provide an expeirnece with fewer animations and transitions to users who have their operating system accessbility preference to reduce motion.
