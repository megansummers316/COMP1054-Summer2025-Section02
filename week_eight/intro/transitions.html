What Are Transitions in CSS?

In CSS, elements have states (link states like hover, for example). By default, elements can swtich between these states instantly, which can be kind of jarring.

Using CSS transitions allow us to create a natural transition between states by providing the opportunity to interpolate between the first or initial state and the element's target or final state. Interpolation means to create "in-between" steps.

Why Do We Use Transitions?

By adding these extra steps, we are able to enhance user experience and interaction. Transtions can help to provide visual direction, confirmation, supports and hints about the cause of the effect of interaction with an element.

In order to add a transition to your element, you must choose an element that can be animated and can have middle steps between their start and final states. You also need an event that will trigger that state change in order to activate the transition. Common events include pseudo-classes like :hover, :focus, :focus-within, :target, :active

Be mindful of performance when adding transitions to your page. For example, when some properties like width and height change they cause other elements to change and move around as well, which can have an impact on the performance of your page.
